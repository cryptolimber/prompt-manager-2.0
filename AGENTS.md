# 项目记忆

本文件用于记录 Codex 在当前项目中必须遵守的项目级约定。

## 适用范围

除非子目录存在更具体的 `AGENTS.md`，否则本文件的规则适用于整个项目。

## 通用原则

- 修改内容必须聚焦用户需求。
- 不覆盖无关的本地改动。
- 新增或修改代码时，优先沿用项目已有约定。
- 行为发生变化且项目存在测试框架时，需要新增或更新测试。
- 引入新依赖前，先检查项目是否已有可用能力或现有依赖。
- 所有描述性和非专业词汇，统一使用中文语言。

## 需求处理原则

- 开始实现前，先分析需求，明确用户真正要完成的目标。
- 对需求进行澄清和整理，识别范围、约束、风险和验收点。
- 需求较复杂时，按功能、文件、阶段或风险拆分为可执行任务。
- 完成需求后，记录本次需求的完成点、关键决策、验证结果和遗留事项。
- 不把临时推测当成项目事实；无法确认的信息需要标记为待确认。

## `PROJECT.md` 使用方法

- `PROJECT.md` 是项目需求档案，用于避免后续工作失忆。
- 每次处理新需求前，先阅读 `PROJECT.md`，了解已有需求记录和项目状态。
- 每次完成需求后，更新 `PROJECT.md` 的需求记录。
- 记录应包含：需求日期、需求内容、需求分析、任务拆分、完成点、验证结果和后续事项。
- 已完成内容只追加或修正事实，不随意删除历史记录。

## 项目环境

- Python 版本使用 3.12。
- 使用 `uv` 管理虚拟环境和依赖。
- Python 项目配置集中在 `pyproject.toml`。
- 本地虚拟环境使用 `.venv`，不提交到项目。
- 初始化或同步环境时运行 `uv sync`。
- 执行 Python 命令时优先使用 `uv run`。

## 项目备注

- 当前项目暂无应用源码文件，已添加 Python 项目环境配置。
- 随着项目结构、命令和约定明确，需要同步更新本文件。
