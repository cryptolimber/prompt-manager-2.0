# Prompt Manager 2.0 打包文档

本文档记录使用 PyInstaller 生成绿色发行包的步骤。绿色发行包不要求目标电脑预装 Python，用户解压后运行可执行文件即可启动服务。

## 结论

PyInstaller 不能在 Linux 下直接打出可在 Windows 上运行的 `.exe`。它基本遵循“在哪个平台打包，就产出哪个平台的可执行文件”：

- Linux 打包：产出 Linux 可执行文件。
- Windows 打包：产出 Windows `.exe`。
- macOS 打包：产出 macOS 可执行文件或 `.app`。

如果要分发给 Windows 用户，需要在 Windows 机器、Windows 虚拟机，或 Windows CI 环境中打包。Linux 下通过 Wine 等方式交叉打包不作为本项目推荐方案，排错成本高且稳定性不可控。

## 目录约定

- 打包产物统一放到项目根目录的 `dist/` 下。
- PyInstaller 临时构建目录放到 `build/` 下。
- `dist/` 中的实际发行包不提交到 Git，仅保留 `dist/.gitkeep` 以保留目录。

## 准备环境

项目使用 Python 3.12 和 `uv` 管理环境。

首次打包前，在目标打包平台执行：

```bash
uv sync --group build
```

如果只是运行源码开发环境：

```bash
uv sync
```

## Windows 打包

在 Windows CMD 中进入项目根目录后执行：

```bat
uv sync --group build
uv run pyinstaller ^
  --noconfirm ^
  --clean ^
  --console ^
  --onedir ^
  --name prompt_manager_2.0 ^
  --distpath dist/windows ^
  --workpath build/pyinstaller ^
  --specpath build/pyinstaller ^
  --collect-all nicegui ^
  main.py
```

打包完成后，产物目录为：

```text
dist/windows/prompt_manager_2.0/
```

用户使用方式：

1. 将 `dist/windows/prompt_manager_2.0/` 整个目录压缩为 zip。
2. 用户解压到任意目录。
3. 双击 `prompt_manager_2.0.exe`。
4. 浏览器访问 `http://localhost:8080`。

Windows 包需要保留控制台窗口，因此打包命令显式使用 `--console`。不要使用 `--windowed` 或 `--noconsole`，否则启动后的黑色终端窗口会被隐藏，运行日志也无法直接显示在终端中。

## macOS 打包

在 macOS 终端中进入项目根目录后执行：

```bash
uv sync --group build
uv run pyinstaller \
  --noconfirm \
  --clean \
  --console \
  --onedir \
  --name prompt_manager_2.0 \
  --distpath dist/macos \
  --workpath build/pyinstaller \
  --specpath build/pyinstaller \
  --collect-all nicegui \
  main.py
```

打包完成后，产物目录为：

```text
dist/macos/prompt_manager_2.0/
```

未签名应用在 macOS 上可能被系统拦截。内部使用时可右键打开；正式对外分发建议做代码签名和 notarization。

## Linux 打包

当前开发环境是 Linux，可以先在本机验证 Linux 包：

```bash
uv sync --group build
uv run pyinstaller \
  --noconfirm \
  --clean \
  --console \
  --onedir \
  --name prompt_manager_2.0 \
  --distpath dist/linux \
  --workpath build/pyinstaller \
  --specpath build/pyinstaller \
  --collect-all nicegui \
  main.py
```

打包完成后运行：

```bash
./dist/linux/prompt_manager_2.0/prompt_manager_2.0
```

## 验证清单

每个平台打包后至少验证：

- 在没有项目 `.venv` 的机器上运行发行包。
- 启动后能访问 `http://localhost:8080`。
- 数据库环境配置可以保存到用户目录的 `PROMPT_MANAGER_CONFIG.json`。
- 本地 SQLite 状态库可以在用户目录 `.prompt_manager_2_0/local_status.db` 创建。
- 能连接真实 MySQL 环境并查看 Prompt 列表。
- 新增、编辑、删除、同步、推送逻辑与源码运行时一致。
- Windows exe 双击启动后会保留黑色控制台窗口，启动日志、NiceGUI/Uvicorn 日志和异常堆栈会输出到该窗口。

## 常见问题

### Linux 打包的结果能否给 Windows 使用

不能。Linux 下打包出来的是 Linux 可执行文件，Windows 不能直接运行。Windows 绿色包需要在 Windows 上打包。

### 为什么使用 `--onedir`

`--onedir` 会生成一个目录，里面包含可执行文件和运行依赖。它更适合作为绿色安装包分发，启动速度也通常比单文件 `--onefile` 更稳定。

### 是否可以使用 `--onefile`

可以，但不作为默认推荐。`--onefile` 启动时需要先解压内部依赖到临时目录，首次启动更慢，且遇到静态资源或杀毒软件拦截时更难排查。

### 端口 8080 被占用怎么办

当前程序默认使用 8080 端口。如果目标机器已经有程序占用该端口，需要先关闭占用进程，或后续改造为可配置端口。

### exe 黑框为什么不能关闭

当前程序是本地 Web 服务，黑色控制台窗口就是服务进程本身。关闭该窗口等同于停止 Prompt Manager 2.0，浏览器页面也会随之不可用。
